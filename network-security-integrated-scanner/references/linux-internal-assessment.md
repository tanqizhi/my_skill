# Linux internal assessment

Support Ubuntu/Debian and RHEL/CentOS/Rocky/Alma families. Use read-only checks for OS and patches, accounts and authentication, SSH, firewall, listeners and services, packages, file permissions, kernel controls, containers, logs, scheduled jobs, startup entries, processes, connections, and compromise indicators.

Use the supplied root account directly for read-only checks. For a normal account, run unprivileged checks first and request confirmation before a required `sudo` command. Do not install packages or write to the target.
